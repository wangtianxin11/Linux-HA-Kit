# MongoDB 运维手册

## 连接

```bash
mongosh --port 27017 -u admin -p 'YOUR_PASSWORD' --authenticationDatabase admin
```

---

## 副本集状态

### 查看节点状态
```bash
mongosh --port 27017 -u admin -p 'YOUR_PASSWORD' --authenticationDatabase admin \
  --eval "rs.status().members.forEach(m => print(m.name, m.stateStr))"
```

### 查看复制延迟
```bash
mongosh --port 27017 -u admin -p 'YOUR_PASSWORD' --authenticationDatabase admin \
  --eval "rs.printSecondaryReplicationInfo()"
```

### 查看副本集配置
```bash
mongosh --port 27017 -u admin -p 'YOUR_PASSWORD' --authenticationDatabase admin \
  --eval "rs.conf()"
```

### 查看节点状态及复制时间戳
```bash
mongosh --port 27017 -u admin -p 'YOUR_PASSWORD' --authenticationDatabase admin \
  --eval "rs.status().members.forEach(m => print(m.name, m.stateStr, 'lag:', m.optimeDate))"
```
> 两节点时间戳一致表示复制延迟为 0

### 查看所有数据库（PRIMARY）
```bash
mongosh --port 27017 -u admin -p 'YOUR_PASSWORD' --authenticationDatabase admin \
  --eval "db.getMongo().getDBNames().forEach(d => print(d))"
```

### 在 SECONDARY 上查看数据库
```bash
mongosh --port 27017 -u admin -p 'YOUR_PASSWORD' --authenticationDatabase admin \
  --eval "rs.secondaryOk(); db.getMongo().getDBNames().forEach(d => print(d))"
```
> ⚠️ SECONDARY 默认不可读，必须加 `rs.secondaryOk()` 才能查询
> ⚠️ 空数据库不会复制到 SECONDARY，需至少有一个集合且有数据才会同步

## 服务管理

```bash
systemctl start mongod
systemctl stop mongod
systemctl restart mongod
systemctl status mongod
```

### 查看日志
```bash
tail -100 /var/log/mongodb/mongod.log
journalctl -u mongod --no-pager | tail -50
```

## 故障排查

### 数据目录权限修复（mongod 启动失败时）
```bash
chown -R mongodb:mongodb /var/lib/mongodb
chown -R mongodb:mongodb /var/log/mongodb
systemctl start mongod
```

### 手动创建管理员（无认证状态）
```bash
# 先停服务，以无认证方式临时启动
systemctl stop mongod
mongod --port 27017 --dbpath /var/lib/mongodb --noauth --fork --logpath /tmp/mongod-tmp.log

# 创建用户
mongosh --port 27017 --eval "
  db.getSiblingDB('admin').createUser({
    user: 'admin',
    pwd: 'YOUR_PASSWORD',
    roles: [{ role: 'root', db: 'admin' }]
  })
"

# 关闭临时进程，恢复正式服务
mongod --port 27017 --dbpath /var/lib/mongodb --shutdown
# 修复目录权限（临时以 root 启动会导致权限变更）
chown -R mongodb:mongodb /var/lib/mongodb
systemctl start mongod
```

### 手动初始化副本集
```bash
mongosh --port 27017 --eval "
  rs.initiate({
    _id: 'rs0',
    members: [
      { _id: 0, host: '192.168.10.106:27017', priority: 2 },
      { _id: 1, host: '192.168.10.109:27017', priority: 1 }
    ]
  })
"
```

### 强制切主（Keepalived 故障转移时自动触发）
```bash
# 在目标主机上执行
bash /etc/keepalived/notify_mongo_master.sh
```

## 配置文件位置

| 文件 | 路径 |
|------|------|
| MongoDB 配置 | `/etc/mongod.conf` |
| 副本集 keyfile | `/etc/mongodb/keyfile` |
| 数据目录 | `/var/lib/mongodb` |
| 日志文件 | `/var/log/mongodb/mongod.log` |
| Keepalived 切主脚本 | `/etc/keepalived/notify_mongo_master.sh` |
| Keepalived 切备脚本 | `/etc/keepalived/notify_mongo_backup.sh` |
