# 热备安装脚本 需求规范

## 背景与目标

运维/开发人员在部署中间件时，需要一套统一的交互式安装脚本，支持单机和热备（主从/双主）两种模式。
脚本主体负责框架调度、配置管理、组件选择；各组件的具体安装逻辑由独立子脚本实现，后续按需扩展。

---

## 用户故事

- 作为开发人员，我希望运行一个主脚本就能选择安装哪些组件，以便不需要手动管理多个安装脚本。
- 作为开发人员，我希望全局 SSH 配置只配置一次并持久保存，以便下次执行无需重复填写。
- 作为开发人员，我希望选择热备模式时脚本自动处理主备两台服务器的部署，以便减少重复操作。

---

## 验收标准

- [ ] Given 首次运行脚本 When 未检测到全局配置文件 Then 引导用户填写主机IP、SSH账号、SSH密码并保存到配置文件
- [ ] Given 再次运行脚本 When 检测到已有全局配置文件 Then 直接展示当前配置并询问是否修改，跳过重新填写
- [ ] Given 用户选择"单机模式" When 选择组件后 Then 仅在当前服务器执行对应组件的安装子脚本
- [ ] Given 用户选择"热备模式" When 选择主从或双主后 Then 同时在主、备两台服务器上执行安装子脚本
- [ ] Given 用户选择安装组件 When 多选后确认 Then 按顺序依次调用各组件安装子脚本
- [ ] Given 各组件安装子脚本存放在约定目录 When 主脚本调用时 Then 能正确找到并执行对应脚本
- [ ] Given 脚本在 Ubuntu 20.04 / 24.04 上运行 When 执行任意流程 Then 兼容两个版本，无报错

---

## 核心交互流程

```
启动主脚本
  │
  ├─ 检查全局配置文件是否存在
  │     ├─ 不存在 → 引导配置 SSH（主机IP、账号、密码）→ 保存配置
  │     └─ 存在   → 展示当前配置 → 询问是否修改
  │
  ├─ 选择部署模式
  │     ├─ 单机
  │     └─ 热备 → 选择拓扑（主从 / 双主）
  │
  ├─ 多选组件（MySQL / Redis / Nacos / MongoDB / Nginx / Keepalived+VIP）
  │
  └─ 逐个调用组件安装子脚本
```

## 配置文件规范

**全局配置文件**路径：`<脚本目录>/conf/global.conf`

```ini
# 全局 SSH 配置（明文保存）
MODE=           # standalone | ha
HA_TOPO=        # master-slave | dual-master（热备时有效）

MASTER_HOST=
SLAVE_HOST=     # 热备时有效

SSH_USER=
SSH_PASS=

INSTALL_COMPONENTS=  # 逗号分隔，如 mysql,redis,nacos
```

## 目录结构约定

```
<脚本根目录>/
├── install.sh              # 主脚本（框架入口）
├── conf/
│   └── global.conf         # 全局配置（首次运行后生成）
├── components/
│   ├── mysql/
│   │   └── install.sh      # MySQL 安装子脚本
│   ├── redis/
│   │   └── install.sh
│   ├── nacos/
│   │   └── install.sh
│   ├── mongodb/
│   │   └── install.sh
│   ├── nginx/
│   │   └── install.sh
│   └── keepalived/
│       └── install.sh
└── packages/               # 离线安装包目录（按需放置）
```

## 边界与约束

- **SSH 执行**：热备模式下，主脚本通过 `sshpass` + SSH 将命令分发到备机执行
- **明文密码**：SSH 密码明文保存在 `conf/global.conf`，建议限制文件权限为 `600`
- **离线/在线**：本规范不限定安装方式，由各组件子脚本自行决定
- **OS 兼容**：目标系统为 Ubuntu 20.04 / 24.04
- **子脚本接口**：主脚本调用子脚本时，通过环境变量或参数传递全局配置（主机、账号、密码、模式）

## MVP 范围

**本期必做（主体框架）：**
- 全局配置的检测、录入、保存、展示
- 部署模式选择（单机 / 热备-主从 / 热备-双主）
- 组件多选菜单
- 按顺序调用各组件子脚本（子脚本暂用 placeholder）
- 目录结构初始化

**后续迭代：**
- 各组件具体安装逻辑（各自独立开发）
- 安装结果汇总报告
- 回滚/卸载功能
